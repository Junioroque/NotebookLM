Uma aplicação web é um sistema de software que permite a interação entre diferentes sistemas e usuários através da internet, utilizando protocolos como HTTP ou HTTPS é diferente de um site estático simples, as aplicações web são dinâmicas, o que significa que o conteúdo pode mudar com base nas informações do usuário ou em dados vindos de um banco de dados, como um saldo bancário ou uma lista de produtos em uma pizzaria.

O funcionamento de uma aplicação web baseia-se em dois pilares principais:
> Lado do Cliente (Client-side): É o que roda diretamente no navegador do usuário (como Chrome ou Edge). Tecnologias como JavaScript ou Blazor WebAssembly são usadas aqui para criar interfaces interativas que podem ser atualizadas sem a necessidade de recarregar a página inteira, como nas chamadas SPAs (Single Page Applications)
> Lado do Servidor (Server-side): É onde o código é processado em um computador remoto (servidor). Em tecnologias como ASP.NET Core, o servidor processa a lógica de negócio, gerencia a segurança e acessa o banco de dados antes de enviar a resposta para o navegador.

Dentro do ecossistema .NET, existem diferentes modelos para construir essas aplicações:
1- MVC (Model-View-Controller): Um padrão de arquitetura que separa a aplicação em três camadas: o modelo de dados (Model), a interface visual (View) e o controlador da lógica (Controller).
2- Razor Pages: Um modelo mais simples e focado em páginas, onde o código C# e o HTML são combinados para gerar conteúdo dinâmico no servidor.
3- Web API: Um modelo que não possui uma interface visual própria; ele funciona como um "garçom" que entrega dados (geralmente em formato JSON) para que outras aplicações (como aplicativos móveis ou sites em React/Angular) os consumam.
4- Blazor: Uma tecnologia moderna que permite aos desenvolvedores escreverem tanto o código do servidor quanto o do cliente utilizando C#, eliminando em muitos casos a necessidade de usar JavaScript diretamente
